## 先搞懂三个概念

把Leo系统想象成一家公司：

| 概念 | 通俗解释 | 类比 |
|------|---------|------|
| **Skills（技能）** | 具体干活的工具 | 公司里的各种软件工具 |
| **Agents（代理）** | 会用工具的员工 | 不同部门的专业员工 |
| **Workflows（工作流）** | 多人协作的流程 | 跨部门协作的SOP |

**关系**：工作流调度员工 → 员工使用工具 → 完成任务

## 二、Agents（代理）- 5个员工

| Agent | 擅长什么 | 激活示例 |
|-------|---------|---------|
| task-agent | 执行具体任务、排版发布 | `帮我排版并发布这篇文章` |
| research-agent | 信息搜索、资料整理 | `帮我调研智慧农贸市场` |
| analysis-agent | 数据分析、趋势预测 | `分析这组销售数据的趋势` |
| creative-agent | 内容创作、文案生成 | `帮我写一篇营销文案` |
| realestate-agent | 房地产业务全流程 | `生成楼盘营销方案` |

### 怎么用Agent？

**方式1：让系统自动选**（推荐）
```
分析宁波房地产市场
```
系统会自动选择最合适的Agent。

**方式2：指定Agent**
```
用research-agent帮我调研AI眼镜市场
```

### Agent vs Skill 区别

- **Skill**：单一功能，比如"排版"
- **Agent**：会思考，能组合多个Skill完成复杂任务

**例子**：
- 用Skill：`帮我排版` → 只做排版
- 用Agent：`帮我写一篇房产分析文章并排版发布` → 自动调用搜索+写作+排版+发布

## 三、Workflows（工作流）- 3条流水线

### 内容生产线（content-pipeline）
**流程**：研究 → 创作 → 发布
```
运行content-pipeline，主题是"2026年宁波楼市分析"
```

### 研究线（research-pipeline）
**流程**：收集 → 分析
```
运行research-pipeline，主题是"AI眼镜竞品分析"
```

### 分析线（analysis-pipeline）
**流程**：分析 → 报告
```
运行analysis-pipeline，分析这组数据
```

## 五、使用技巧

### 技巧1：说清楚需求
| 不好 | 好 |
|-----|-----|
| 帮我做个东西 | 生成项目营销手册 |
| 帮我排版 | 帮我排版，用数据驱动型风格 |

### 技巧2：复杂任务用工作流
| 简单任务 | 复杂任务 |
|---------|---------|
| 直接说需求 | 运行xxx工作流 |
| 帮我排版 | 运行内容生产线 |

### 技巧3：不确定用什么？
直接说你要做什么，系统会自动选择：
```
我要写一篇关于宁波楼市的分析文章
```

## 六、文档自动更新

当系统新增了技能、代理或工作流后，本指南会自动更新。

**如何触发更新**：
直接告诉Claude：
```
更新入门指南
```

Claude会自动扫描系统配置并更新本文档。

**配置文件位置**：
- 所有Skills、Agents、Workflows的配置都在：`leo-config/settings/config.yaml`
- 新增组件后，只需在该文件中添加配置，然后说"更新入门指南"即可
